# 1단계: 메인 화면 기본 요구사항 및 중복 가입 방어

우리 프로젝트의 메인 화면 파일인 `src/app/page.tsx`를 아래 요구사항과 제공된 전체 소스코드를 바탕으로 '네가 직접 수정'해줘. 내가 직접 복사 붙여넣기 하지 않고 네가 파일에 반영해 주길 원해.

[🔧 기능 및 디자인 요구사항]
1. **전체 화면 와이드 풀 배경**: 기존 모바일 크기 제한을 없애고 PC 모니터 화면 전체에 웅장한 스타디움 노을빛 아우라 그라데이션이 흐르게 해줘.
2. **우측 상단 프로필 모달**: 우측 상단 프로필 아이콘을 누르면 플레이어 이름을 입력할 수 있는 반투명 글래스모피즘 모달창(팝업)이 뜨게 하고, 저장 시 로컬스토리지 및 Supabase `leaderboard` 테이블에 등록되게 해줘.
3. **실시간 연승 랭킹(리더보드)**: 메인 메뉴 카드 아래에 Supabase `leaderboard` 테이블에서 최고 연승자 상위 5명을 가져와 보여주는 실시간 순위표 섹션을 이쁘게 만들어줘.
4. **기존 기능 유지**: Supabase 선수 명단(players) 불러오기, 속도/슈팅/점프 아이콘 바인딩 로직은 절대 깨뜨리지 마.
5. **중복 가입 방어 로직**: 데이터 삽입 시 이미 존재하는 닉네임 에러(code: 23505)가 발생하면, 즉시 에러를 뿜지 말고 기존 유저의 `max_streak`와 `current_streak` 정보를 테이블에서 가져와 로컬 스토리지와 동기화 시키는 똑똑한 로그인 분기문 코드를 완성해줘.

# 2단계: 게임 설명서 팝업 모달 개편 및 게스트 모드

게임 설명서 팝업 모달을 직관적인 게이머 감성으로 개편해줘.
1. 내부 탭 스위치 버튼들의 명칭을 "🎮 게임 방법 & 조작법", "👤 프로필 설정 & 연승 랭킹"으로 친근하게 교체해줘.
2. 불필요하게 괄호 처리되어 있던 영문 표기(Current Streak, Max Streak)들을 과감히 삭제해서 가독성을 높이고, 화면 하단 레이아웃을 겹치게 하던 중복 팁 컴포넌트들을 지워줘.
3. 데이터베이스 등록 프로세스가 부담스러운 학우들을 위해 "😎 게스트로 플레이하기" 버튼을 만들고, 이 버튼을 누르면 로컬 저장소에 '게스트'로 식별값을 심은 뒤 메인 화면으로 자동 진입하게 설계해줘.
4. 유저명이 '게스트'인 상태에서는 하단 리더보드 연동을 스킵하며, 실제 대두축구 게임 종료 시 "게스트 모드이므로 연승 기록이 데이터베이스에 세이브되지 않습니다"라는 경고 배너가 출력되도록 연결해줘.

# 3단계: 애플리케이션 배경 및 헤더 레이아웃 밸런스 튜닝

우리 전체 애플리케이션의 배경과 레이아웃 밸런스를 튜닝해줘.

# 4단계: AI 제미나이 스쿼드 메이커 아키텍처 및 안전 설계

우리 프로젝트에 대망의 'AI 제미나이 스쿼드 메이커' 기능을 완벽하게 구현해줘. API 키 보호를 위해 프론트와 백엔드 분리 아키텍처를 적용해야 해.
1. **백엔드 라우트 생성 (`src/app/api/analyze/route.ts`)**: 최신 프리미엄 패키지인 `@google/genai` 라이브러리를 임포트하고 `GoogleGenAI` 인스턴스를 생성해줘. 모델은 가장 효율적이고 속도가 강력한 최신 `gemini-3.1-flash-lite` 또는 `gemini-3.5-flash`를 매핑해줘.
2. **구글 AQ 키 마침표 버그 해결**: 구글 AI 스튜디오에서 발급된 `AQ.`로 시작하는 신규 키는 특수문자 마침표로 인해 REST 엔드포인트 URL 파싱 오류(404 Not Found)를 유발할 수 있으니, 반드시 공식 인스턴스 옵션 내에 `apiKey` 변수로 바인딩하거나 HTTP 호출 시 `x-goog-api-key` 헤더 아티팩트에 명시적으로 탑재되도록 안전 설계해줘.
3. **프롬프트 페이로드 엔지니어링**: 제미나이에게 "너는 인천대학교 스마트 대시보드 분석가 또는 월드컵 전술 전문가다"라는 페르소나를 완벽히 인입하고, 사용자가 입력한 필터 정보와 선수단 스탯(속도, 슈팅, 점프) 통계를 종합 파싱하여 총평 및 마크다운 리포트를 뽑아내도록 바인딩 코드를 짜줘.

# 5단계: 모달 및 UI 톤앤매너 디테일 고도화

설명서 모달과 스쿼드 선택 UI의 디테일을 한 차원 더 높은 수준으로 다듬어줘.
1. 기존 설명서의 2가지 설명서 시스템을 **3개 가지설명서**로 업그레이드하여, 세 번째 설명서에 "🤖 AI 스쿼드 메이커 가이드"를 정식 편입시켜줘. 여기에는 단과대 필터링, 이수구분 분석, 스쿼드 시너지 측정 요령을 상세히 설명해줘.
2. 선수단 초이스 그리드 화면 상단에 박혀있어 시선을 분산시키던 하드코딩된 중복 안내 문자열 "선택 시 해당 포지션 슬롯에 자동 매칭됩니다." 구문을 흔적도 없이 삭제해줘.

# 8단계: 미니게임 물리 엔진 스케일 및 경기장 규격 재조정

우리 대두축구 미니게임 화면(`src/app/game/page.tsx`)의 물리 엔진 스케일성과 경기장(Canvas) 규격을 아케이드 손맛에 맞게 전면 재조정해줘.

[📐 1. 경기장 크기 및 골대 스케일링 사양]
- **캔버스 비율 고정**: 경기장 전체 캔버스 해상도를 가로 1024px, 세로 576px (16:9 황금 비율)로 명시적으로 고정하고, 반응형으로 축소되더라도 물리 좌표가 깨지지 않게 뷰박스를 격리해줘.
- **골대(Goalpost) 규격 조정**: 좌측과 우측 끝에 배치되는 골대의 높이를 세로 캔버스 기준 하단에서 35% 높이(약 200px)로 설정하고, 골대 상단 크로스바에 공이 맞았을 때 리얼하게 튕겨 나가는 원형 물리 충돌체(Circle Collider)를 심어줘.
- **캐릭터 및 공의 스케일**: 대두 캐릭터의 히트박스 반지름은 45px, 축구공의 반지름은 18px로 조정하여 캐릭터 머리 크기 대비 공이 너무 작거나 크지 않게 시각적 밸런스를 맞춰줘.

[⚖️ 2. 중력 및 마찰력 계수 (Physics Coefficients)]
- **공의 중력 및 탄성**: 축구공의 중력 계수를 기존보다 1.2배 높여서 묵직하게 가라앉게 하고, 탄성 계수(Restitution)를 0.75로 세팅하여 땅에 튕겼을 때 통통 튀는 쫀득한 탄력을 구현해줘.
- **캐릭터 점프 제한**: 캐릭터가 점프했을 때 캔버스 천장(Top: 0)을 뚫고 올라가지 않도록 Y축 상단 경계 충돌 구문을 확실하게 추가하고, 바닥에 착지할 때 미끄러지지 않도록 지면 마찰력을 0.2로 인입해줘.

기존에 연결해 둔 키보드 조작(방향키, 스페이스바) 핸들러와 슈팅 파워 수식은 이 물리 스케일 안에서 정상 작동하도록 값들을 유기적으로 보정해 줘야 해.

# 9단계: AI 분석가 페르소나 및 통계 분석 로직 고도화

`src/app/api/analyze/route.ts`에 탑재된 구글 제미나이 호출 소스코드를 수정하여 AI의 전술 분석 지능과 페르소나 수준을 최고 등급으로 업그레이드해줘.

[🧠 1. AI 전술 분석가 페르소나 강제 주입]
- 너는 세계 최고 권위의 스포츠 통계학자이자, 유럽 챔피언스리그 우승을 경험한 베테랑 전술 감독(예: 펩 과르디올라)이다. 칭찬만 하는 챗봇 말투를 버리고, 지극히 데이터 중심적이며 때로는 날카롭고 비판적인 해설가 톤앤매너를 유지해라.

[📊 2. 능력치 수학적 분석 로직 가이드]
- 유저가 선택한 4명 선수 스탯의 '평균값'과 '표준편차'를 머릿속으로 계산해라.
- 만약 [속도] 평균이 85 이상이면 "초고속 역습형 스쿼드", [슈팅] 평균이 85 이상이면 "중거리 화력 집중형", [점프] 평균이 85 이상이면 "공중볼 장악 및 세트피스 특화팀"으로 전술 테마를 명확히 정의해라.
- 국가(Nation) 데이터를 체크하여 4명 중 2명 이상이 같은 나라이거나 같은 대륙이면 "조직력(Chemistry) 보너스 가중치"를 분석글에 반영해라.

[📋 3. 정형화된 출력 포맷 강제]
반드시 아래의 마크다운 형식을 엄격히 준수하여 리포트를 리턴해줘:
1. **[종합 평가 티어]**: S+, S, A, B, C, D 중 냉정하게 산출된 하나의 티어 배지 표기
2. **[팀 전술 성향]**: 1줄 핵심 요약
3. **[공격 및 수비 시너지 진단]**: 계산된 스탯 근거를 제시하며 구체적으로 서술 (예: "이 팀은 수비수의 점프력이 낮아 롱볼 축구에 치명적인 약점을 노출합니다.")
4. **[명장의 한마디]**: 이 스쿼드의 약점을 보완하기 위한 전술적 조언 1가지

텍스트 글자 수가 너무 짧지 않게, 최소 300자 이상의 깊이 있는 장문 분석 보고서가 나오도록 프롬프트 템플릿을 고도화해줘.

# 10단계: 그리드 내 선수 카드 이름 겹침 현상 최종 픽스

메인 화면의 월드컵 스타 선수 명단 그리드에서 심각한 시각적 버그가 발견됐어. 해상도가 줄어들거나 폰 화면으로 보면, 카드 우측 상단 절대 좌표(`absolute`)로 박혀있는 포지션 텍스트 배지(FW, GK 등) 밑으로 선수 이름 글자가 파고들어가서 텍스트가 겹치고 잘려 보여.

[🔧 해결 지시사항]
- 선수 카드의 내부 레이아웃 구조를 수정해줘. 국기 원형 아바타의 크기를 `w-14`에서 `w-12 h-12`로 콤팩트하게 축소해서 텍스트가 서 있을 수 있는 가로 공간을 최대한 넓혀줘.
- 선수 이름과 국가 정보가 적히는 텍스트 박스 컨테이너에 우측 패딩 값(`pr-8` 또는 `pr-10`)을 강제적으로 부여해줘.
- 이 패딩 안심 영역 덕분에 이름이 아무리 길어지거나 가로 폭이 좁아지더라도 글자가 우측 끝 배지와 절대 물리적으로 부딪히지 않고, 자연스럽게 말줄임표(`truncate`) 처리가 되거나 아랫줄로 개행되도록 레이아웃 정렬을 완벽하게 픽스해줘.
